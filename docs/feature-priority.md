# 기능 우선순위

## 1. 목적

본 문서는 개발 과정에서 기능 범위가 과도하게 확장되는 것을 방지하고,  
공모전 제출 시점까지 가장 중요한 기능부터 안정적으로 완성하기 위해 작성한다.

우선순위는 다음 기준으로 판단한다.

- 핵심 가치 전달 가능 여부
- 데모 시연 중요도
- 구현 난이도 대비 효과
- 사용자 체감 가치

---

## 2. 우선순위 기준

### P0
반드시 구현해야 하는 핵심 기능  
이 기능이 없으면 서비스의 정체성이 드러나지 않는다.

### P1
핵심 가치를 강화하는 기능  
시간 내 구현 가능하면 반드시 포함하는 것이 좋다.

### P2
있으면 좋지만 초기 공모전 버전에서는 제외 가능  
시간이 남으면 구현한다.

---

## 3. P0 기능

### 공통
- 역할 기반 접근(강사/수강생)

### 강사용
- 강의 생성
- 세션 생성
- PDF 업로드
- 녹화본 업로드

### AI 처리
- PDF 텍스트 추출
- 목차 추출
- 페이지별 주제문 생성
- STT 자막 생성
- 강의자료 기반 QA
- 기본 퀴즈 생성

### 수강생용
- 강의 신청
- 세션 진입
- PDF 열람
- 영상 + 자막 시청
- 챗봇 질문
- 퀴즈 풀이

### UX 핵심
- 목차 클릭 시 관련 페이지 이동
- 영상 재생 중 현재 목차 표시
- 챗봇 답변에 출처 표시

---

## 4. P1 기능

### 강사용
- 목차/매핑 결과 수동 수정
- 자막 결과 검토 화면
- 퀴즈 문항 수정 기능
- 자료/세션 공개 여부 설정

### 수강생용
- 자막 클릭 시 해당 타임스탬프 이동
- 목차별 퀴즈 제공
- 진도율 표시
- 오답 기반 재학습 링크

### AI 고도화
- 강의자료 기반 자막 전문용어 후보정
- 자료 + 자막 복합 검색
- 출처 페이지/목차 정확도 개선

---

## 5. P2 기능

### 강사용
- 과제 생성 및 제출 관리
- 수강생별 학습 현황 대시보드
- 많이 질문된 주제 분석
- 콘텐츠 재업로드 버전 관리

### 수강생용
- 개인 맞춤 복습 추천
- 질문 히스토리 정리
- 북마크/메모 기능
- 학습 리포트 제공

### 시스템/운영
- 운영자 전용 대시보드
- 알림 기능
- 다중 강사/조교 권한
- 모바일 최적화
- 실시간 강의 연동

---

## 6. 개발 순서 제안

### 1단계
- 강의/세션 CRUD
- PDF 업로드
- PDF 분석 결과 저장 및 조회

### 2단계
- 녹화본 업로드
- STT 및 자막 저장
- 목차-타임스탬프 연결

### 3단계
- 수강생 학습 화면
- PDF 뷰어
- 영상 플레이어
- 자막 표시
- 목차 네비게이션

### 4단계
- 챗봇 QA
- 출처 표시
- 퀴즈 자동 생성/풀이

### 5단계
- 품질 개선
- UI 다듬기
- 예외 처리
- 데모 시나리오 고정

---

## 7. 범위 통제 원칙

다음 원칙을 지키며 개발한다.

1. P0가 완성되기 전에는 P1 기능을 확장하지 않는다.
2. P1은 데모에 효과가 큰 기능 위주로 선택한다.
3. P2는 절대 필수로 간주하지 않는다.
4. "기능 수"보다 "핵심 흐름 완성도"를 우선한다.
5. 발표에서 설명하기 어려운 기능은 우선순위를 낮춘다.
6. 로컬에서만 되는 기능보다 배포 환경에서 안정적으로 동작하는 기능을 우선한다.
7. 공모전 제출 시 라이브 URL에서 바로 시연 가능한 흐름만 우선 구현한다.
