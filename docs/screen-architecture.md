# 화면 구조 설계

## 1. 목적

본 문서는 서비스의 주요 화면과 각 화면의 역할, 핵심 UI 요소를 정의한다.  
초기 MVP에서는 실제 사용자 흐름과 데모 시나리오를 기준으로 화면을 설계한다.

---

## 2. 전체 화면 구조

### 공통 화면
- 메인 대시보드
- 강의 목록 화면
- 강의 상세 화면

### 강사용 화면
- 강의 생성/수정 화면
- 세션 관리 화면
- 세션 상세 화면
- 자료 업로드 화면
- 자료 분석 결과 화면
- 녹화본 업로드 화면
- 자막/매핑 검토 화면
- 퀴즈 관리 화면

### 수강생용 화면
- 수강 강의 목록 화면
- 세션 학습 화면
- PDF 뷰어 화면
- 영상 플레이어 화면
- 챗봇 패널
- 퀴즈 풀이 화면
- 학습 결과 화면

---

## 3. 화면별 상세 설계

## 3-1. 초기 진입 팝업
### 목적
로그인 기능 없이, 강사 / 수강생 둘 중 하나 선택

## 3-2. 강의 목록 화면
### 목적
강의 탐색 및 진입

### 주요 요소
- 상단 검색바
- 강의 카드 리스트
- 강의명, 강사명, 기간, 정원 표시
- 강의 생성 버튼(강사만)
- 내가 수강 중인 강의 / 전체 강의 탭

---

## 3-3. 강의 상세 화면
### 목적
강의 정보 확인 및 세션 진입

### 주요 요소
- 강의 제목, 설명, 일정
- 세션 리스트
- 강의 신청 버튼(수강생)
- 세션 추가 버튼(강사)
- 수강생 수, 공개 상태 표시

## 3-4. 세션 상세 화면
### 목적
세션별 자료와 녹화본 상태를 확인하고 다음 학습 화면으로 진입

### 주요 요소
- 세션 제목, 공개 상태
- 자료 목록과 처리 상태
- 녹화본 목록과 STT 상태
- 강사용 자료 업로드 / 녹화본 업로드 CTA
- 강의 상세로 돌아가기 링크

---

## 3-5. 강의 생성/수정 화면
### 목적
강의 메타데이터 입력 및 수정

### 주요 요소
- 강의명
- 강의 설명
- 시작일 / 종료일
- 정원
- 썸네일 업로드
- 저장 버튼

---

## 3-6. 세션 관리 화면
### 목적
세션 생성 및 순서 관리

### 주요 요소
- 세션 리스트
- 세션 추가 버튼
- 세션 순서 변경
- 세션 공개/비공개 토글
- 세션명 / 일정 / 상태 표시

---

## 3-7. 자료 업로드 화면
### 목적
PDF 강의자료 등록

### 주요 요소
- 파일 업로드 영역
- 자료 유형 선택
- 정보 추출 여부 체크
- 업로드 버튼
- 업로드된 자료 목록
- 처리 상태 배지

---

## 3-8. 자료 분석 결과 화면
### 목적
PDF 구조화 결과 확인

### 주요 요소
- 좌측: 목차 패널
- 중앙: PDF 뷰어
- 우측: 페이지 주제문 / 키워드 / 요약 패널
- 목차별 시작/종료 페이지 정보
- 수정 버튼
- 공개 버튼

### 핵심 UX
- 목차 클릭 시 해당 페이지 이동
- 현재 페이지가 어떤 목차에 속하는지 표시

---

## 3-9. 녹화본 업로드 화면
### 목적
세션 녹화본 등록 및 상태 확인

### 주요 요소
- 영상 업로드
- 영상 목록
- STT 처리 상태
- 자막 생성 완료 여부
- 세션 연결 정보

---

## 3-10. 자막/매핑 검토 화면
### 목적
STT 결과 및 목차 연결 상태 확인

### 주요 요소
- 영상 플레이어
- 자막 리스트
- 현재 구간 타임스탬프
- 연결된 목차 표시
- 자막 수정 또는 검토 UI
- 공개 버튼

---

## 3-11. 세션 학습 화면
### 목적
수강생의 핵심 학습 화면

### 주요 구성
- 좌측: 목차/세션 네비게이션
- 중앙: 영상 플레이어 또는 PDF 뷰어
- 우측: 챗봇 / 퀴즈 / 요약 패널
- 상단: 세션 제목, 진행 상태

### 핵심 UX
- 자료, 영상, 챗봇, 퀴즈가 한 화면 맥락 안에서 연결되어야 한다.
- 사용자가 목차를 중심으로 이동할 수 있어야 한다.

---

## 3-12. PDF 뷰어 영역
### 목적
강의자료 열람

### 주요 요소
- 페이지 이동
- 확대/축소
- 현재 페이지 표시
- 페이지별 주제문 표시
- 목차 클릭 시 페이지 이동

---

## 3-13. 영상 플레이어 영역
### 목적
녹화본 시청 및 자막 기반 탐색

### 주요 요소
- 재생/일시정지
- 타임라인
- 자막 on/off
- 현재 목차 표시
- 자막 클릭 이동
- 목차 클릭 시 해당 시점 이동

---

## 3-14. 챗봇 패널
### 목적
강의자료 기반 질의응답 제공

### 주요 요소
- 질문 입력창
- 대화 리스트
- 답변 카드
- 출처 표시(페이지 번호, 목차명)
- 출처 클릭 이동 기능

### 핵심 UX
- 답변은 항상 강의자료 기반이어야 한다.
- 출처 정보가 명확히 보이도록 한다.

---

## 3-15. 퀴즈 풀이 화면
### 목적
복습 및 이해도 점검

### 주요 요소
- 문항 리스트
- 객관식/단답형 입력 UI
- 제출 버튼
- 결과 확인
- 해설 및 출처 표시

---

## 3-16. 학습 결과 화면
### 목적
퀴즈 결과 및 복습 포인트 제공

### 주요 요소
- 점수
- 정답/오답 표시
- 해설
- 관련 목차/페이지 이동 링크

---

## 5. 대표 레이아웃 제안

### 세션 학습 화면 레이아웃
- **좌측 사이드바**: 세션 목차 / 자료 구조 / 퀴즈 진입
- **중앙 메인 영역**: PDF 뷰어 또는 영상 플레이어
- **우측 패널**: 챗봇, 페이지 요약, 퀴즈, 추천 학습 정보

이 화면이 본 프로젝트의 핵심 화면이며,  
발표 시연에서도 가장 많이 활용되는 대표 화면이 된다.

## 6. UI ↔ API 매핑 (핵심)

아래는 화면 액션과 매핑된 최소한의 API 계약 요약입니다. 각 화면은 해당 엔드포인트를 호출하거나 결과를 폴링하여 상태를 표시합니다.

- **자료 업로드 화면**
	1. 프론트: `POST /api/upload-urls` → 업로드용 `uploadUrl` 수신
	2. 프론트: signed URL로 파일 업로드(스토리지)
	3. 프론트: `POST /api/materials` (메타 생성)
	4. 프론트: `GET /api/materials/{materialId}` 폴링(권장 5초) → `processingStatus` 표시

- **자료 분석 결과 화면**
	- 프론트: `GET /api/materials/{materialId}/sections` → 목차 표시
	- 프론트: `GET /api/materials/{materialId}/pages` → 페이지별 주제문/요약 표시

- **녹화본 업로드 화면**
	1. 프론트: `POST /api/upload-urls` → signed URL 수신
	2. 프론트: 파일 업로드 → `POST /api/recordings` (메타 생성)
	3. 프론트: `GET /api/recordings/{recordingId}` 폴링(권장 5초) → `sttStatus` 표시
	4. STT 완료 시 `subtitleUrl`을 받아 플레이어에 연결

- **자막/매핑 검토 화면**
	- 프론트: `GET /api/recordings/{recordingId}/segments?sectionId=&pageNumber=&cursor=&limit=` → 세그먼트 목록/매핑 조회
	- 프론트: 수정 제출 시 `PATCH /api/recordings/{recordingId}/segments/{segmentId}`

- **세션 학습 화면 (수강생)**
	- 초기 로드: `GET /api/sessions/{sessionId}` (세션 메타 + materials + recordings 목록)
	- 목차 클릭: 클라이언트 측 내비게이션 + `GET /api/materials/{materialId}/pages` 또는 `GET /api/recordings/{recordingId}/subtitle` 호출

- **챗봇 패널**
	- 사용자가 질문 입력 시: `POST /api/qa` (동기)
	- 응답에는 `answer` 및 `sourceRefs` 포함 → 클릭 시 해당 자료/타임스탬프로 이동

- **퀴즈 관리/풀이**
	- 강사(생성, 동기 소량): `POST /api/sessions/{sessionId}/quizzes` → draft 생성
	- 강사(생성, 비동기 권장): `POST /api/sessions/{sessionId}/quiz-generations` → `GET /api/quiz-generations/{jobId}`로 상태 조회
	- 강사(공개): `POST /api/quizzes/{quizId}/publish`
	- 수강생(응시): `POST /api/quizzes/{quizId}/attempts` → 제출 및 채점

폴링 권장 지점
- 자료/녹화본 업로드 후 처리 상태 확인: `GET /api/materials/{id}`, `GET /api/recordings/{id}` (권장 5초)
- 실시간 데이터(채팅 streaming 등)는 MVP에서 제외

보안/데모
- MVP 데모 모드: 무인증 환경에서는 `role` 쿼리 파라미터로 화면 진입 역할을 전환하도록 지원

위 매핑을 기준으로 엔드포인트 상세 스펙(요청/응답 필드, 오류 코드)을 `docs/planning.md`에 초안으로 추가했습니다.
