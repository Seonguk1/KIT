
# 유저 플로우

## 1. 개요

본 서비스는 크게 두 사용자 그룹을 중심으로 동작한다.

- 강사
- 수강생

각 사용자는 서로 다른 목적과 행동 흐름을 가지며,  
서비스는 이 두 흐름이 자연스럽게 연결되도록 설계한다.

> 참고: MVP 데모는 무인증 데모 모드로 제공될 수 있으며, 데모 환경에서는 `로그인` 단계가 생략됩니다. 데모에서는 역할 선택 모달에서 강사/수강생을 고른 뒤 화면 분기를 적용하고, 선택값은 브라우저 로컬 저장소에 유지해 재진입 시에도 동일 역할 흐름을 이어갈 수 있습니다. 또한 대시보드/상세 화면의 `역할 다시 선택` 액션으로 언제든 역할을 전환할 수 있습니다.

## 2. 강사 유저 플로우

### 2-1. 강의 생성
1. 강사가 로그인한다.
2. 강의 목록 화면으로 이동한다.
3. 새 강의 생성 버튼을 클릭한다.
4. 강의명, 설명, 시작일, 종료일, 정원 등을 입력한다.
5. 강의를 생성하면 강의 상세 화면으로 자동 이동한다.

### 2-2. 세션 생성
1. 강의 상세 화면으로 진입한다.
2. 세션 추가 버튼을 클릭한다.
3. 세션명, 일정, 공개 여부를 설정한다.
4. 세션을 저장한다.

### 2-3. 세션 상세 확인
1. 강의 상세 화면의 세션 목록에서 원하는 세션을 선택한다.
2. 세션 상세 화면에서 자료와 녹화본 상태를 확인한다.
3. 강사 모드에서는 상단 CTA로 자료 업로드/녹화본 업로드를 바로 시작한다.
4. 필요한 경우 강의 상세 화면으로 돌아간다.

### 2-4. 강의자료 업로드
1. 세션 상세 화면에서 강의자료 업로드를 선택한다.
2. PDF 파일을 업로드한다.
3. 자료 유형과 정보 추출 여부를 선택한다.
4. 참고: 업로드 완료 시 자료 레코드가 생성되고 상태는 `uploaded` → `queued`(워커 폴링 5초) → `processing` → `completed`/`failed` 로 전이합니다. UI는 `queued` 상태를 폴링해 처리 진행을 표시합니다.
5. 업로드 완료 후 처리 상태를 확인한다.

### 2-5. PDF 정보 추출 결과 확인
1. 시스템이 PDF를 분석한다.
2. 목차, 페이지별 주제문, 키워드, 페이지 매핑이 생성된다.
3. 강사는 분석 결과를 검토한다.
4. 필요 시 목차명 또는 매핑 정보를 수정한다.
5. 자료 공개 여부를 설정한다.

### 2-6. 녹화본 업로드
1. 세션 상세 화면에서 녹화본 업로드를 선택한다.
2. 녹화본 파일을 업로드한다.
3. 시스템이 STT를 수행한다.
4. 참고: 녹화본 업로드 시 `recordings` 레코드가 생성되고 상태는 `uploaded` → `queued`(워커 폴링 5초) → `transcribing` → `post_processing` → `completed`/`failed` 로 전이합니다. STT/후처리 실패 시 재시도 정책(최대 3회, 지수 백오프)을 따릅니다.
5. 강의자료를 참고하여 자막 후보정을 진행한다.
6. 타임스탬프와 목차 매핑 결과를 생성한다.
7. 강사는 자막 결과를 확인하고 공개 여부를 설정한다.

### 2-7. 퀴즈 생성 및 관리
1. 세션 또는 목차 단위로 퀴즈 생성 버튼을 클릭한다.
2. 시스템이 강의자료를 바탕으로 문항을 생성한다.
3. 강사는 문항과 정답을 검토/수정한다.
4. 퀴즈를 공개한다.

### 2-8. 강의 운영
1. 수강생 목록을 확인한다.
2. 수강생별 학습 현황을 확인한다.
3. 수강생들이 많이 질문한 주제나 어려워한 파트를 파악한다.
4. 다음 세션 자료 또는 퀴즈를 보완한다.

## 3. 수강생 유저 플로우

### 3-1. 강의 탐색 및 신청
1. 수강생이 로그인한다.
2. 강의 목록 화면에서 강의를 탐색한다.
3. 원하는 강의를 선택한다.
4. 강의 상세 정보를 확인한다.
5. 수강 신청을 진행한다.

### 3-2. 세션 학습 진입
1. 수강 중인 강의 목록에서 강의를 선택한다.
2. 강의 상세 화면에서 세션 목록을 확인한다.
3. 공개된 세션을 선택한다.
4. 세션 상세 화면에서 자료와 녹화본 상태를 확인한다.
5. 세션 학습 화면으로 진입한다.

### 3-3. 강의자료 학습
1. 세션 내 업로드된 PDF 자료를 연다.
2. 좌측 또는 상단의 목차 패널을 확인한다.
3. 원하는 목차를 클릭하여 해당 페이지로 이동한다.
4. 페이지 요약 또는 핵심 키워드를 참고한다.

### 3-4. 녹화본 시청
1. 세션 내 녹화본 플레이어를 실행한다.
2. 자막을 켜고 시청한다.
3. 현재 재생 중인 타임스탬프가 어떤 목차인지 확인한다.
4. 목차를 클릭해 해당 구간으로 이동한다.
5. 자막 구간을 클릭해 재생 위치를 이동한다.

### 3-5. 챗봇 질의응답
1. 학습 중 챗봇 패널을 연다.
2. 질문을 입력한다.
3. 시스템이 동기적으로 임베딩 검색(material_pages, transcript_segments)을 수행하고 상위 컨텍스트를 결합한 뒤 LLM 호출로 답변을 생성해 즉시 반환한다. 답변에는 출처(page/section/timestamp)를 명시한다.
4. 자료 범위를 벗어나는 질문에 대해서는 범위 초과 안내를 포함한 제한 응답을 반환한다.
5. 필요한 경우 출처를 클릭해 자료로 이동한다.

### 3-6. 퀴즈 풀이 및 복습
1. 세션 또는 목차 단위 퀴즈를 선택한다.
2. 문항을 풀고 제출한다.
3. 정답 및 해설을 확인한다.
4. 틀린 문항의 출처 목차/페이지를 다시 학습한다.

## 4. 핵심 사용자 여정

### 강사의 핵심 여정
강의 생성 → 세션 생성 → 자료 업로드 → 정보 추출 확인 → 녹화본 업로드 → 자막/목차 매핑 생성 → 퀴즈 공개

### 수강생의 핵심 여정
강의 신청 → 세션 진입 → 자료 열람 → 영상 시청 → 챗봇 질문 → 퀴즈 풀이 → 복습

---

## 5. 핵심 UX 포인트

### 강사 측
- 업로드 후 AI가 자동으로 구조화해주는 경험이 중요하다.
- 복잡한 설정보다 "업로드 → 결과 확인" 흐름이 직관적이어야 한다.
- 자료와 영상의 연결 상태를 한눈에 확인할 수 있어야 한다.

### 수강생 측
- 자료, 영상, 자막, 챗봇, 퀴즈가 분리되지 않고 한 학습 맥락 안에서 이어져야 한다.
- 목차를 중심으로 탐색이 가능해야 한다.
- 질문에 대한 답변은 반드시 근거와 함께 제시되어야 한다.

## 6. 대표 시나리오

### 시나리오 1: 강사의 수업 준비
- 강사는 이번 주차 강의자료 PDF와 녹화본을 업로드한다.
- 시스템이 자동으로 목차를 추출하고 페이지를 정리한다.
- 녹화본 자막이 생성되고 목차와 연결된다.
- 강사는 생성된 퀴즈를 검토한 뒤 수강생에게 공개한다.

### 시나리오 2: 수강생의 시험 전 복습
- 수강생은 세션 학습 화면에서 "Transformer 구조" 목차를 선택한다.
- 관련 PDF 페이지와 영상 구간으로 이동한다.
- 자막을 보며 내용을 복습한다.
- 챗봇에게 추가 질문을 하고, 출처 페이지를 확인한다.
- 해당 목차 퀴즈를 풀며 이해도를 점검한다.
